# 上級問題の4択を相手レンジの総コンボ数に応じた数値に変更する

## Why

上級問題の4択は現在、固定境界（10 / 40 / 80）によるコンボ数帯（バケット）だが、これは前回の指示の誤解によるもの。本来の意図は「自分のハンドが相手のレンジに対してどのくらい強いのかを大まかに掴めているか」を問うことであり、固定の帯では相手レンジの総コンボ数（デッドカード除外後）との相対関係が問えない。例えば総コンボ数 150 のレンジに対しては「3 / 27 / 67 / 129」のように、0〜総コンボ数の全域に分散した具体的な数値を選択肢にすべき。

## What Changes

- **BREAKING（仕様変更）**: 上級問題の4択を「固定境界のコンボ数帯ラベル」から「相手レンジの総コンボ数に応じて分散させた具体的な数値4つ」に変更する
  - 正解は実際の負けコンボ数そのもの。誤答3つは、総コンボ数を比率で分割したゾーンのうち正解が属さない3ゾーンから1つずつ生成し、選択肢が 0〜総コンボ数の全域に大きく分散するようにする
  - ゾーンの比率境界は設定ファイルに集約し、マジックナンバーを散らさない（現行方針を踏襲）
  - 選択肢は昇順で表示する（数値が問題ごとに変わるため「位置固定の反射訓練」の意図は帯方式から引き継がない）
- 正解ゾーンの偏り防止（目標ゾーンを選んで再抽選するロジック）は、固定帯ではなく「その候補の総コンボ数から導出したゾーン」を基準に判定するよう変更する
- 固定バケット関連の設定値・導出ロジック（`ADVANCED_CHOICE_BUCKET_BOUNDS` / `ADVANCED_CHOICE_BUCKETS` / `advancedBucketIndex`）は新方式の設定・ロジックに置き換える
- README の上級編・4択の記述を新仕様に更新する
- 解説・リザルト画面で正確な負けコンボ数を表示する挙動は変更しない。序盤ヒント（負けセル二重ハイライト）も変更しない

## Capabilities

### New Capabilities

（なし）

### Modified Capabilities

- `question-engine`: 「上級問題のバケット選択肢」要件を「総コンボ数に応じて分散させた数値4択」要件に置き換える。上級問題の生成要件のうち、目標バケットによる再抽選をゾーン基準に改める
- `game-screens`: 「4択ボタンのラベル表示」要件のうち、上級のラベルを帯（例:「11〜40」）から数値に変更する。昇順表示は維持する

## Impact

- `src/config/gameConfig.ts`: バケット境界・帯導出・帯判定を、ゾーン比率境界とゾーン導出・判定に置き換え
- `src/lib/questions.ts`: `bucketChoices` を数値4択の生成（正解+ゾーン別誤答）に置き換え、`makeAdvancedQuestion` の目標帯再抽選をゾーン基準に変更
- `src/lib/__tests__/gameConfig.test.ts` / `src/lib/__tests__/questions.test.ts`: バケット関連テストの置き換え
- `src/pages`（プレイ画面）: 選択肢ラベルは問題データ駆動のため原則変更なし（確認のみ）
- `README.md`: 上級編の4択仕様の記述を更新
