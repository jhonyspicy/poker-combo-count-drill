# debug-level-start

## Purpose

開発時のみ有効な、任意レベルからのプレイ開始機能を定義する。デバッグ効率のための機能であり、本番のゲームフロー・記録には影響を与えない。

## ADDED Requirements

### Requirement: 開発時のみのデバッグボタン表示

トップページは、開発ビルド（`import.meta.env.DEV` が真）のときのみ、Lv1〜Lv5 それぞれからプレイを開始できるデバッグボタン群を表示する SHALL。本番ビルドではデバッグボタンを表示せず、バンドルにも含めない SHALL。

#### Scenario: 開発サーバーでの表示

- **WHEN** `npm run dev` で起動したアプリのトップページを表示する
- **THEN** Lv1〜Lv5 の各レベルで開始できるデバッグボタン群が表示される

#### Scenario: 本番ビルドでの非表示

- **WHEN** `npm run build` した本番ビルドのトップページを表示する
- **THEN** デバッグボタン群は表示されない

#### Scenario: ボタンにレベルとタイプが示される

- **WHEN** デバッグボタン群が表示される
- **THEN** 各ボタンにレベル番号と対応する問題タイプ名（基礎・レンジ表・フロップ・ターン・リバー）が示される

### Requirement: 指定レベルからのプレイ開始

デバッグボタンでプレイを開始したとき、プレイ画面は指定されたレベルから開始する SHALL。Lv2 以上で開始した場合、初回問題はそのレベルに対応する問題タイプで動的生成され、制限時間もそのレベル・タイプに応じた値になる SHALL。開始レベルの指定がない遷移（通常スタート・リトライ）は従来どおり Lv1 から開始する SHALL。

#### Scenario: Lv3 デバッグ開始

- **WHEN** トップページで Lv3 のデバッグボタンをタップする
- **THEN** プレイ画面が Lv3・フロップ問題・Lv3 相当の制限時間で開始される

#### Scenario: Lv1 デバッグ開始

- **WHEN** トップページで Lv1 のデバッグボタンをタップする
- **THEN** 通常スタートと同一の条件（単純問題キューの先頭から）でプレイが開始される

#### Scenario: 通常スタートへの非干渉

- **WHEN** 通常のスタートボタンでプレイを開始する
- **THEN** 従来どおり Lv1 の単純問題から開始される

#### Scenario: デバッグ開始後のレベル進行

- **WHEN** Lv2 以上でデバッグ開始し、そのレベルの習熟条件を達成する
- **THEN** 通常プレイと同様にレベルが 1 上がる

### Requirement: デバッグランの自己ベスト非記録

Lv2 以上から開始したラン（デバッグラン）では、ゲームオーバー時・時間切れ時ともに自己ベストを更新しない SHALL。デバッグラン中にレベルアップしてもデバッグラン扱いは変わらない SHALL。Lv1 から開始したランは通常プレイと同様に記録する。

#### Scenario: デバッグランのゲームオーバー

- **WHEN** Lv2 以上でデバッグ開始したランで誤答または時間切れになる
- **THEN** localStorage の自己ベストは変更されず、リザルト画面に自己ベスト更新演出は表示されない

#### Scenario: デバッグランで高スコアを出した場合

- **WHEN** デバッグランで既存の自己ベストを上回る連続正解数を出してゲームオーバーになる
- **THEN** 自己ベストは更新されない

#### Scenario: デバッグラン後のリトライ

- **WHEN** デバッグラン終了後のリザルト画面でリトライボタンをタップする
- **THEN** 通常の Lv1 プレイとして開始され、以降の記録は通常どおり行われる
