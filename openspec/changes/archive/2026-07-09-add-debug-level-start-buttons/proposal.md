# デバッグ用レベル開始ボタンの追加

## Why

現在プレイは必ず Lv1（単純問題の全問クリア）から始まるため、Lv2 以降（レンジ表・フロップ等）の動作確認をするたびに Lv1 を突破する必要があり、開発・デバッグの効率が悪い。任意のレベルから即座にプレイを開始できる導線が必要。

## What Changes

- トップページに、各レベル（Lv1〜Lv5）から直接プレイを開始できるデバッグ用ボタン群を追加する
- デバッグボタンは **開発時（`import.meta.env.DEV`）のみ表示**し、GitHub Pages の本番ビルドには含めない
- プレイ画面が開始レベルを受け取れるようにする（指定がなければ従来どおり Lv1 開始）
- **デバッグ開始（Lv2 以上から開始）したプレイは自己ベストを記録しない**（正規プレイと条件が異なるため記録を汚染しない）

## Capabilities

### New Capabilities

- `debug-level-start`: 開発時のみ有効な、任意レベルからのプレイ開始機能。トップページのデバッグボタン、プレイ画面の開始レベル受け取り、デバッグプレイ時の自己ベスト記録スキップを扱う。

### Modified Capabilities

（なし — 既存の `game-screens` / `score-tracker` の要件は本番動作としては変わらない。デバッグ時の挙動は新設の `debug-level-start` スペックで規定する）

## Impact

- `src/pages/TopPage.tsx`: 開発時のみ表示するデバッグボタン群を追加
- `src/pages/PlayPage.tsx`: location state から開始レベルを受け取り、初期状態（レベル・初回問題・制限時間）を切り替える。デバッグ開始時は自己ベスト更新をスキップ
- `src/pages/ResultPage.tsx`: デバッグプレイのリトライ動線に影響がないか確認（必要ならデバッグフラグの引き継ぎ）
- サーバー・依存関係・ビルド設定への影響なし
