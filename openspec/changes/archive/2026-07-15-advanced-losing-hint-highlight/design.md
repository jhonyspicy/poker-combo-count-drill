# 設計: 上級編 序盤ヒント（負けセルの二重ハイライト）

## Context

上級問題（range-vs-board）は `makeAdvancedQuestion()`（src/lib/questions.ts）が生成し、正解計算の中核は `countRangeVsHero()`。現状は `expandPresetCombos()` でレンジ全体をフラットに展開して勝敗を数えるため、「どのハンドタイプに負けコンボが含まれるか」の情報は捨てられている。

レンジ表の描画は `RangeGrid`（169セルの `RangeCell[]` を受け取り `hit` でハイライト）。プレイ画面（PlayPage）は `score`（連続正解数）を持ち、問題の通し番号 = `score`（0始まり）と一致する。リザルト画面はレンジ表を描画しないため影響しない。

## Goals / Non-Goals

**Goals:**

- 上級の序盤 N 問（初期値 10、`gameConfig.ts` で調整可能）で、負けコンボを含むセルを二重ハイライトする
- ヒントの終了が予告なしに見えないよう、残りヒント数を表示する
- 負け判定はロジックで算出する（ハードコードしない）。テストで検証する

**Non-Goals:**

- 準上級レベルの新設（難易度は3つのまま）
- 中級・初級への影響（レンジ表の既存表示は変えない）
- リザルト画面へのヒント表示（解説文で十分）
- ヒント有無での自己ベストの区別（スコア形式は変えない）

## Decisions

### 1. 負けセルの算出は `countRangeVsHero` を拡張する

`expandPresetCombos` のフラット展開をやめ、`preset.hands` ごとに `expandHandCombos` で展開して勝敗を数え、負けコンボが1つ以上あるハンドタイプのラベル集合 `loseHands: Set<string>` を `RangeVsHeroCount` に追加で返す。

- 理由: 勝敗評価のループは1箇所に保つ（別関数で二重に評価すると計算が2倍になり、判定のズレも起きうる）。合計値の計算結果は従来と同一なので既存テストで回帰を検証できる
- 代替案: 別関数 `findLosingHands()` を新設 → 評価ロジックの重複と実行コスト増で却下

### 2. ヒント情報は問題データに常に含め、表示の判定はプレイ画面が行う

`RangeCell` に任意プロパティ `lose?: boolean` を追加し、`buildAdvancedQuestion` が `loseHands` と `cellLabel` の一致でセルに印を付ける（プリセットのハンド表記と `cellLabel` の表記は同一形式）。PlayPage が `question.type === 'range-vs-board' && score < ADVANCED_HINT_QUESTIONS` のときだけ `RangeGrid` に表示フラグを渡す。

- 理由: 問題生成を純粋（難易度のみ入力）に保ち、`generateQuestion` のシグネチャを変えない。表示の出し分けという UI 関心事は UI 側に置く
- 代替案: `generateQuestion(difficulty, streak)` にして生成時に埋め込む → 生成とプレイ進行が結合し、テストも書きにくくなるため却下

### 3. 描画は `RangeGrid` に `showLoseHint` プロパティを追加

`showLoseHint` が true のとき、`hit && lose` のセルを負けセル色で塗り、凡例に「負けあり」を追加する。false のときは従来表示（後方互換。初級・中級・ヒント終了後の上級は無変更）。

- 負けセル色は `uiConfig.ts` の `RANGE_GRID_COLORS` に追加（`loseBg` / `loseFg`）。既存のアンバー（#f5b83d）と明確に区別できる赤系にする
- 凡例文言は「負けあり」とする（= このセルに負けコンボが1つ以上含まれる。全コンボが負けとは限らないため「負け」と断定しない）

### 4. 残りヒント数はレンジ表パネルの直下に表示

PlayPage で、ヒント有効中のみ「ヒント表示中（残り N 問）」の小さな一行をレンジ表の下に出す。残り = `ADVANCED_HINT_QUESTIONS - score`。

- 理由: ヘッダーは難易度名とスコアで埋まっており、フィードバック行は正解/不正解表示と競合する。ヒントの対象物（レンジ表）の直近に置くのが視線移動も少ない
- 最後の1問で「残り 1 問」と表示されるため、次問からの消滅は予告済みになる

### 5. 設定は `ADVANCED_HINT_QUESTIONS = 10` を gameConfig.ts に追加

既存の `ADVANCED_ANSWER_MIN/MAX` と並べて定数として置く（difficulty-modes の設定一元管理の要件に従う）。

## Risks / Trade-offs

- [ヒント終了直後（11問目）の体感難度が急上昇する] → 残りヒント数の表示で予告する。それでも急なら N の調整やヒントの段階的縮退（将来）で対応。まず定数1つで様子を見る
- [「負けあり」セルの意味の誤解（セル全体が負けだと思う）] → 凡例文言を「負けあり」とし、遊び方画面に「ハイライトはそのセルに負けコンボが1つ以上含まれる印。全部が負けとは限らない」と明記する
- [打ち切りフォールバックで負けコンボ0の問題が出た場合、ヒント表示中なのに負けセルが1つもない] → 仕様上答えも 0 なので整合はしている（極めて稀）。表示は「負けセルなし」のまま出す
- [過去の自己ベストより記録が伸びやすくなる] → 許容する（proposal 記載）。スコアの互換性自体は保たれる
