# 上級編 序盤ヒント（負けセルの二重ハイライト）

## Why

上級編は「負けているハンドタイプの特定」と「そのコンボ数の計算」の2段階を同時に要求するため、難しすぎて学習の入口として機能していない。序盤の問題に限り「負けているコンボを含むセル」をレンジ表上で追加ハイライトすることで、まず計算の練習に集中させ、慣れた頃に自力での特定を要求する段階的な難易度設計にする。

## What Changes

- 上級問題の生成時に、レンジ内の各セル（ハンドタイプ）について「自分より強いコンボを1つ以上含むか」を算出し、問題データに持たせる
- プレイ画面は、連続正解数が設定値（初期値 10）未満の間、上級問題のレンジ表で「負けセル」を通常のレンジ内ハイライトと区別できる色で二重ハイライトする
- ヒントが有効な間は「ヒントは残り○問」の表示を添え、ヒント終了が突然に見えないようにする
- ヒント有効問題数は `gameConfig.ts` の定数として一元管理する（準上級レベルの新設はしない。難易度は3つのまま）
- レンジ表パネルの凡例に「負けセル」の項目を追加する（ヒント表示中のみ）
- 遊び方画面と README に、ハイライト＝「そのセルに負けコンボが1つ以上含まれる」という意味（16コンボ全部が負けとは限らない）を明記する

## Capabilities

### New Capabilities

（なし）

### Modified Capabilities

- `question-engine`: 上級問題の生成要件に「負けコンボを含むハンドタイプ（セル）の算出と問題データへの付与」を追加
- `game-screens`: プレイ画面の要件に「上級序盤の負けセル二重ハイライト・残りヒント数表示・凡例追加」を、遊び方画面の要件に「ヒントの意味の説明」を追加
- `difficulty-modes`: 設定一元管理の対象に「上級のヒント有効問題数」を追加

## Impact

- `src/config/gameConfig.ts` — ヒント有効問題数の定数を追加
- `src/lib/questions.ts` — `countRangeVsHero` 周辺でセル別の負け判定を算出し、上級問題データに含める
- `src/lib/rangeQuestions.ts` — `RangeCell` に負けセルフラグ（任意プロパティ）を追加
- `src/components/RangeGrid.tsx` — 負けセルの描画と凡例の出し分け
- `src/config/uiConfig.ts` — 負けセルのハイライト色を追加
- `src/pages/` — プレイ画面（ヒント適用判定・残りヒント数表示）、遊び方画面（説明追記）
- `src/lib/__tests__/` — セル別負け判定のテスト追加
- `README.md` — 上級編の仕様に序盤ヒントを追記
- 既存の自己ベストとの互換性: スコア形式は変わらないが、序盤が易しくなるため過去記録より伸びやすくなる（許容する）
