# 中級編のハンドをオープンレンジからピックアップする

## Why

中級編（board-count）は現在、「あなたのハンド」を完全ランダムに配り、出題対象の「相手のハンド」も固定ランク集合（A〜7）からランダムに選んでいる。そのため、出題ハンドがボード・自分のハンドと無関係になることが多く、デッドカードが1枚も絡まない問題（答えがそのまま 6 / 4 / 12 になる問題）が頻出する。これではデッドカードを考慮したコンボカウントの練習にならない。

## What Changes

- 中級編の「あなたのハンド」を、ランダムな2枚ではなく**オープンレンジのハンドタイプから選んで具体化**する
- 中級編の出題対象ハンド（「相手が ○○○ を持っているコンボ数は？」の ○○○）を、**オープンレンジ内のハンドタイプから選ぶ**ようにする
- 出題対象ハンドは、**ボードまたは自分のハンドとランクが重なる（デッドカードが1枚以上絡む）ハンドに限定**する。これによりデッドカード考慮が必ず答えに影響する
- 出題対象ハンドの残りコンボ数が 0 の場合は出題しない（1 以上を保証）
- README の中級編の説明を新仕様に合わせて更新する

## Capabilities

### New Capabilities

（なし）

### Modified Capabilities

- `question-engine`: 中級問題（デッドカード問題）の生成要件を変更する。自分のハンド・出題対象ハンドをオープンレンジから選出し、出題対象ハンドにデッドカードが1枚以上絡むことを保証する

## Impact

- `src/lib/questions.ts` — 中級問題生成（`makeIntermediateQuestion` と周辺関数）の抽選ロジックを変更
- `src/lib/presetRanges.ts` — プリセットを id で取得するヘルパー等の追加（必要に応じて）
- `src/config/gameConfig.ts` — 中級編で使うレンジ id を設定として集約（必要に応じて）
- `src/lib/__tests__/questions.test.ts` — 中級問題のテストを新要件（レンジ内選出・デッドカード関連性）に合わせて追加・更新
- `README.md` — 中級編の説明を更新
- UI コンポーネントへの影響はなし（問題データの構造は変わらない）
