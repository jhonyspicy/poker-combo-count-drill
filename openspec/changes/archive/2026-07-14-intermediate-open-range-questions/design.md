# 設計: 中級編のハンドをオープンレンジからピックアップする

## Context

中級問題の生成は `src/lib/questions.ts` の `makeIntermediateQuestion()` が担う。現状のフロー:

1. ストリートを抽選し、シャッフルしたデッキから **ボードと自分のハンドを完全ランダム**に配る（`dealCards`）
2. ハンドカテゴリ（pair / suited / offsuit）を抽選する
3. 出題対象ランクを固定集合 `QUERY_RANKS`（A〜7）から**デッドカードと無関係に**ランダムに選ぶ
4. 答えが 0 のときだけ再抽選する

このため「見えているカードに ○ はなし → 答えはそのまま 6 / 4 / 12」という、デッドカード考慮が不要な問題が高頻度で出る。

プリセットレンジは `src/lib/presetRanges.ts` に定義済みで、`expandHandCombos(hand, dead)`（ハンドタイプ→デッドカード除外の具体コンボ展開）と `parseHand` が既にある。上級問題はこれらを利用している。

## Goals / Non-Goals

**Goals:**

- 中級編の「あなたのハンド」をオープンレンジのハンドタイプから選んで具体化する
- 出題対象ハンド（相手のハンド）をオープンレンジから選び、**必ずデッドカードが1枚以上絡む**（＝素の 6 / 4 / 12 から必ず減る）ことを保証する
- 答え（残りコンボ数）が 1 以上であることを保証する
- 既存の問題データ構造（`Question`）・UI・解説文の形式は維持する

**Non-Goals:**

- 初級・上級の出題ロジックの変更
- 中級編の問題文フォーマットや画面表示の変更（「相手のレンジはオープンレンジ」といった文言追加はしない）
- オープンレンジ以外のプリセット（コールレンジ・3ベットレンジ）の中級編への導入（設定で差し替え可能にはするが、初期値はオープンレンジ固定）

## Decisions

### D1: 自分のハンドを先に決め、その後ボードを配る

オープンレンジからハンドタイプを1つ選び、スートをランダムに割り当てて具体カード2枚に展開してから、残り50枚のデッキからボードを配る。

- 理由: ボードを先に配ってから「レンジ内のハンドになるまで」リジェクションサンプリングするより単純で、必ず1回で確定する
- スート割当: ペアは C(4,2)=6 通り、スーテッドは 4 通り、オフスートは 12 通りから一様ランダム

### D2: 出題対象ハンドは「レンジ内 かつ デッドカードが絡む かつ 残り1コンボ以上」から選ぶ

デッドカード（ボード + 自分のハンド）を固定した上で、オープンレンジの全ハンドタイプを次の条件でフィルタする:

1. **デッド関連性**: そのハンドタイプを構成するランクのカードがデッドカードに1枚以上含まれる
2. **成立性**: `expandHandCombos(hand, dead).length >= 1`

条件1により答えは必ず素のコンボ数（ペア6 / スーテッド4 / オフスート12）より小さくなる（ペアのランクが1枚死ねば C(3,2)=3、スーテッドは該当スートのコンボが死に、オフスートは3コンボ以上減るため）。つまり「デッドカードを数えないと解けない問題」だけが出る。

自分のハンドがオープンレンジ由来（高ランク主体）なので、自分のハンドのランクを含むレンジ内ハンドが常に存在し、候補は実質空にならない。

### D3: カテゴリの重み付けは現行を踏襲する

候補ハンドをカテゴリ分けし、現行の `pickHandCategory`（pair 25% / suited 25% / offsuit 50%）と同じ重みでカテゴリを選んでから、そのカテゴリ内の候補から一様に選ぶ。候補が空のカテゴリは抽選から除外する。

- 理由: オープンレンジはスーテッドが24タイプと過半を占めるため、一様抽選だと「引き算」の練習になるオフスート問題（最重要）が激減する。現行の出題バランスを維持する

### D4: 再抽選とフォールバック

候補が空になるのは理論上ほぼ不可能だが、コードとしては決定的に終了させる:

- 候補が空なら ボード・自分のハンドごと再抽選（現行と同じく上限10回）
- 上限到達時はデッド関連性の条件を外し、「レンジ内 かつ 残り1コンボ以上」のハンドから選ぶ（デッドカード7枚に対し残り45枚あるため必ず存在する）

現行の `aliveRank` フォールバック（答えが必ず6のペア問題）は「デッドカードが絡まない問題」を意図的に出すものなので削除する。

### D5: 使用レンジは設定に集約する

`src/config/gameConfig.ts` に中級編で使うレンジ id（初期値 `'open-range'`）を定数として置き、`presetRanges.ts` に id からプリセットを引くヘルパー `getPresetRange(id)` を追加する。CLAUDE.md の「難易度パラメータは1つの設定ファイルに集約」の方針に従い、将来コールレンジ等へ差し替えられるようにする。

### D6: 既存の問題ビルダーを流用する

`makePairQuestion` / `makeSuitedQuestion` / `makeOffsuitQuestion` は「ランクを内部で抽選する」構造を「選ばれたハンドタイプを受け取る」構造に変更する（`makePairQuestion` の `forceRank` 引数を必須化する方向の一般化）。答えの計算・解説文・誤答生成はそのまま使う。`QUERY_RANKS` と `pickRank` / `pickRankPair` は不要になるため削除する。

## Risks / Trade-offs

- **[出題の偏り]** 自分のハンドとランクが重なる問題が中心になり、ボード由来のデッドカードより自分のハンド由来が多くなる可能性 → デッド関連性は「ボードまたは自分のハンド」のどちらでも満たすため、ボードに高ランクが落ちれば自然にボード絡みの問題も出る。プレイ感を見て将来調整（Non-Goal）
- **[低ランク問題の消失]** 現行 `QUERY_RANKS` は A〜7 だったが、オープンレンジのハンドは 2〜A のランクを含む（A2s 等）。一方 66 以下のペア等は出題されなくなる → レンジ練習という本来の目的に合致するため許容する
- **[テストの書き換え]** 中級テストのうち「ハンド表記は高いランクが先」等は維持されるが、出題対象の分布に依存するテストは新仕様に合わせて更新が必要 → tasks で明示する
