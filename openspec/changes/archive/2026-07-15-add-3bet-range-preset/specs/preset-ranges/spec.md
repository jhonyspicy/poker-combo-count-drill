## MODIFIED Requirements

### Requirement: プリセットレンジの定義
相手のレンジはあらかじめ定義されたプリセット（表示名「オープンレンジ」「コールレンジ」「3ベットレンジ」の3種）から選択する SHALL。問題ごとにレンジを動的生成してはならない SHALL NOT。各プリセットは id・表示名・ハンド表記の列挙（"AA" / "AKs" / "AKo" 形式）で `src/lib/presetRanges.ts` に定義する SHALL。

「オープンレンジ」（id: `open-range`）の内容は以下のハンド集合とする SHALL（合計216コンボ）:

- ペア: AA, KK, QQ, JJ, TT, 99, 88, 77
- スーテッド: AKs, AQs, AJs, ATs, A9s, A8s, A7s, A6s, A5s, A4s, A3s, A2s, KQs, KJs, KTs, K9s, K8s, K7s, K6s, K5s, QJs, QTs, Q9s, JTs
- オフスート: AKo, AQo, AJo, ATo, KQo, KJo

「コールレンジ」（id: `call-range`）の内容は以下のハンド集合とする SHALL（合計118コンボ）:

- ペア: TT, 99, 88, 77, 66, 55, 44
- スーテッド: AQs, AJs, ATs, A9s, A8s, A7s, A5s, A4s, KQs, KJs, KTs, K9s, QJs, QTs, JTs, T9s
- オフスート: AJo

「3ベットレンジ」（id: `three-bet-range`）の内容は以下のハンド集合とする SHALL（合計160コンボ）:

- ペア: AA, KK, QQ, JJ
- スーテッド: AKs, A7s, A6s, A5s, A4s, A3s, A2s, K9s, K8s, K7s, K6s, K5s, K4s, QJs, QTs, Q9s
- オフスート: AKo, AQo, AJo, KQo, KJo, QJo

#### Scenario: プリセットからの選択
- **WHEN** プリセットレンジを使う問題が生成される
- **THEN** 定義済みプリセットのいずれかがランダムに選ばれ、その内容は上記のハンド集合と完全に一致する

#### Scenario: 表示名
- **WHEN** プリセットレンジを使う問題の問題文を生成する
- **THEN** 選ばれたプリセットに応じてレンジ名として「オープンレンジ」「コールレンジ」「3ベットレンジ」のいずれかが表示される

#### Scenario: 表記からグリッドへの展開
- **WHEN** プリセットのハンド表記列を 13×13 グリッドに展開する
- **THEN** ペアは対角セル、スーテッドは右上セル、オフスートは左下セルに正しくマップされる

#### Scenario: オープンレンジのデッドカードなし合計コンボ数
- **WHEN** デッドカードなしでオープンレンジの合計コンボ数を計算する
- **THEN** ペア8種×6 + スーテッド24種×4 + オフスート6種×12 = 216 となる

#### Scenario: コールレンジのデッドカードなし合計コンボ数
- **WHEN** デッドカードなしでコールレンジの合計コンボ数を計算する
- **THEN** ペア7種×6 + スーテッド16種×4 + オフスート1種×12 = 118 となる

#### Scenario: 3ベットレンジのデッドカードなし合計コンボ数
- **WHEN** デッドカードなしで3ベットレンジの合計コンボ数を計算する
- **THEN** ペア4種×6 + スーテッド16種×4 + オフスート6種×12 = 160 となる
