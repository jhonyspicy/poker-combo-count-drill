# Design: add-call-range-preset

## Context

プリセットレンジは `src/lib/presetRanges.ts` の `PRESET_RANGES` 配列に定義され、`pickPresetRange()` が配列からランダムに1つ選ぶ。現在は「オープンレンジ」1件のみ。問題生成（`questions.ts` の初級プリセットレンジ問題・上級レンジ×ボード問題）はこの関数経由でレンジを取得し、コンボ数は常にロジック（`presetComboCount`）で計算される。

ユーザー提供のレンジ表（13×13グリッドの緑セル）を「コールレンジ」として追加する。

## Goals / Non-Goals

**Goals:**
- 「コールレンジ」（118コンボ）を `PRESET_RANGES` に追加し、出題ローテーションに入れる
- 定義の正しさをテストで担保する（表記の正当性・重複なし・合計コンボ数118）
- README のプリセットレンジ記述を更新する

**Non-Goals:**
- レンジ選択UI（ユーザーがレンジを選ぶ機能）は追加しない
- 出題比率の調整（両プリセットは等確率のランダム選択のまま）
- 問題生成ロジック・難易度設定の変更

## Decisions

- **データ追加のみで実装する**: `pickPresetRange()` は既に `PRESET_RANGES` からランダム選択する実装のため、配列にエントリを追加するだけで両問題タイプに反映される。ロジック変更は不要。
- **ハンド集合はレンジ表の緑セルをそのまま転記する**: ペア7種（TT〜44）、スーテッド16種、オフスート1種（AJo）。合計は定義にハードコードせず、既存の `presetComboCount` で計算される（CLAUDE.md の方針どおり）。
- **hands の記述順は既存プリセットに合わせる**: ペア → スーテッド（ランク順）→ オフスートの順で列挙し、可読性を保つ。
- **テストは既存テストのパターンを踏襲する**: `presetRanges.test.ts` に、コールレンジの合計コンボ数（118 = 7×6 + 16×4 + 1×12）と、既存の全プリセット検証（表記正当性・重複なし）がコールレンジにも適用されることを確認するテストを追加する。

## Risks / Trade-offs

- [レンジ表の転記ミス] → スペックにハンド集合を明記し、テストで合計コンボ数118を検証する。実装時にレンジ表の緑セルと突き合わせて検算する
- [プリセットが増えることで問題の難易度が不均一になる] → コールレンジ（118）はオープンレンジ（216）よりコンボ数が少ないが、答えは常にロジック計算なので正解基準に影響なし。難易度チューニングは必要なら別変更で対応
