## Context

全4画面に Tailwind の `px-6`（左右パディング）と `max-w-lg mx-auto`（幅上限と中央揃え）が適用されており、スマホ画面の端に余白が生まれている。モバイルファーストの方針に従い、画面幅を最大限に使う。

## Goals / Non-Goals

**Goals:**
- 全4画面の横方向余白制限を除去してフルワイド表示にする

**Non-Goals:**
- 要素の内側パディング（ボタン・カード等の `p-*`）の変更
- タイポグラフィ・配色・縦方向レイアウトの変更

## Decisions

### Tailwind クラスの直接削除

各ページコンポーネントから `px-6`・`max-w-lg`・`mx-auto` を削除する。抽象化（レイアウトコンポーネント化等）は行わない。変更箇所が少なく、将来的に画面ごとに異なるレイアウトを持つ可能性もあるため。

## Risks / Trade-offs

- **タブレット・PC での表示が横に広がりすぎる** → 現時点ではモバイル専用アプリとして割り切る。将来必要になれば画面幅別の対応を検討する
