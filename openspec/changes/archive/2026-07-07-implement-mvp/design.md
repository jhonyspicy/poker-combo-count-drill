## Context

アプリは Vite + React 19 + TypeScript + Tailwind CSS v4 + React Router v7 (HashRouter) のスタックで構築済みだが、現在 Hello World の状態。READMEに詳細な仕様が定義されており、これを実装する。外部サーバー・API・ログインは持たない。GitHub Pages でホスティングするため HashRouter を維持する。

## Goals / Non-Goals

**Goals:**
- READMEに定義されたMVP仕様（4画面・クイズロジック・レベル進行・自己ベスト保存）を実装する
- コンボ計算ロジックの正確性を確保する（CLAUDE.mdのドメイン知識に準拠）
- モバイルファーストのダーク基調UIを実装する

**Non-Goals:**
- サーバー・API・ランキング機能
- v1以降の拡張候補（順位当て問題・レンジ vs コンボ・SNS共有・i18n）
- ライトモード

## Decisions

### ゲーム状態管理: React useState + URL遷移

外部ステート管理ライブラリ（Zustand等）は導入しない。ゲーム状態（スコア・レベル・現在の問題）は Play 画面のローカル state で管理し、リザルト画面へは `react-router` の `state` で渡す。シンプルな一方向フローで十分。

代替案: Context API でグローバル管理 → 画面数が少なく過剰。

### コンボ計算: 純粋関数モジュール

`src/lib/comboCalculator.ts` に副作用のない純粋関数を集約する。入力はハンドタイプ（ペア/スーテッド/オフスート）とデッドカード配列。計算は CLAUDE.md の定義（AK − AKs = AKo の導出順）に厳密に従う。Vitest でユニットテストを書く。

### タイマー実装: Date.now() 差分

`setInterval` のコールバックでは経過時間を `Date.now()` の差分で計算する（カウント変数を使わない）。モバイルでタブがバックグラウンドになった場合のドリフトを防ぐ。

### 難易度設定: 単一設定ファイル

`src/config/gameConfig.ts` に全パラメータ（1レベルの問題数・基準時間・短縮係数・レベル別出題タイプ構成）を集約する。コンポーネントにマジックナンバーを散らさない。

### 誤答生成: ルールベース

デッドカード問題の誤答は「デッドカードを無視した正解」「スーテッド/オフスートを混同した値」「±1〜3の近傍値」から生成する。乱数シードは不要。

## Risks / Trade-offs

- **コンボ計算バグ** → Vitest でエッジケース（デッドカードが多い・同ランクのデッドカード等）を網羅的にテストする
- **タイマーの UX** → 残り時間が少なくなったときの視覚フィードバック（プログレスバーまたは色変化）を実装しないとユーザーが気付かない可能性がある
- **localStorage のデータ破損** → 読み込み時に型ガードを入れ、不正なデータは無視して自己ベストなし状態として扱う
