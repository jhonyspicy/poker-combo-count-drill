## ADDED Requirements

### Requirement: 4画面ルーティング
HashRouter により `/`（トップ）、`/play`（プレイ）、`/result`（リザルト）、`/howto`（遊び方）の4ルートを提供する SHALL。

#### Scenario: トップページへのアクセス
- **WHEN** ユーザーがアプリを開く
- **THEN** トップ画面が表示される

#### Scenario: 存在しないルート
- **WHEN** ユーザーが未定義のパスにアクセスする
- **THEN** トップ画面にリダイレクトされる

### Requirement: トップ画面
タイトルブロック・自己ベストカード（または初回案内文）・スタートボタン・遊び方へのリンクを縦一列に表示する SHALL。

#### Scenario: 自己ベストがない場合
- **WHEN** localStorage に自己ベストが存在しない状態でトップ画面を表示する
- **THEN** 自己ベストカードの代わりに初回訪問向けの案内文を表示する

#### Scenario: 自己ベストがある場合
- **WHEN** localStorage に自己ベストが存在する状態でトップ画面を表示する
- **THEN** 最高連続正解数・到達レベル・達成日を表示するカードを表示する

#### Scenario: スタートボタンタップ
- **WHEN** ユーザーがスタートボタンをタップする
- **THEN** プレイ画面に遷移する

### Requirement: プレイ画面
問題文・4択ボタン・残り時間インジケーターを表示し、ゲームを進行する SHALL。

#### Scenario: 正解選択
- **WHEN** ユーザーが正解の選択肢をタップする
- **THEN** スコアが +1 され、次の問題が即座に表示される

#### Scenario: 不正解選択
- **WHEN** ユーザーが誤答の選択肢をタップする
- **THEN** ゲームオーバーとなりリザルト画面へ遷移する

#### Scenario: 時間切れ
- **WHEN** 残り時間が 0 になる
- **THEN** ゲームオーバーとなりリザルト画面へ遷移する

### Requirement: リザルト画面
今回のスコア・ゲームオーバーになった問題の正解と解説・自己ベスト（更新演出あり）・リトライ／トップへのボタンを表示する SHALL。

#### Scenario: 自己ベスト更新
- **WHEN** 今回のスコアが自己ベストを上回る
- **THEN** 自己ベスト更新の演出を表示し、localStorage に新記録を保存する

#### Scenario: リトライ
- **WHEN** ユーザーがリトライボタンをタップする
- **THEN** プレイ画面に遷移し、スコア・レベルがリセットされる

### Requirement: 遊び方画面
ゲームルール説明とコンボ表（1326 / 78 / 312 / 936 等）のチートシートを表示する SHALL。

#### Scenario: 遊び方へのアクセス
- **WHEN** ユーザーが遊び方リンクをタップする
- **THEN** 遊び方画面が表示される
