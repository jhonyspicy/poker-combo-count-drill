# CLAUDE.md

ポーカーのコンボ数カウントを鍛えるクイズアプリ（スマホ向けWebアプリ）。
仕様の詳細（ゲームルール・問題タイプ・レベル設計）は README.md を正とする。実装が README と食い違う場合は指摘すること。

## コマンド

```bash
npm run dev      # 開発サーバー
npm run build    # tsc -b && vite build（型チェック込み）
npm run lint     # ESLint
```

変更後は最低限 `npm run build` が通ることを確認する。テストフレームワークは未導入。コンボ計算などのロジックにテストを追加する場合は Vitest を導入すること。

## 技術スタックと制約

- Vite + React 19 + TypeScript + Tailwind CSS v4（`@tailwindcss/vite` プラグイン使用。`tailwind.config` は存在しない）
- React Router v7 の **HashRouter** を使用（GitHub Pages のためパスルーティング不可。変更しないこと）
- `vite.config.ts` の `base: '/poker-combo-count-drill/'` は GitHub Pages 用。アセット参照はこれを前提にする
- main へ push すると GitHub Actions（.github/workflows/deploy.yml）で自動デプロイされる
- **サーバー・API・ログインは持たない**。永続化は localStorage のみ（自己ベストスコア）
- ソースコード中のコメントは日本語を使う
- gitのリモートリポジトリへの更新は絶対にやらないでください。オーナーが手動でプッシュします。

## 設計方針

- **モバイルファースト**。UIはスマホの縦画面（タップ操作）を基準に作る
- **ダーク基調で固定**（ライトモード非対応）。ポーカールームらしい暗めの配色
- 難易度パラメータ（1レベルの問題数、タイプ別基準時間、時間短縮係数、レベル別の出題タイプ構成）は**1つの設定ファイルに集約**し、チューニングしやすくする。マジックナンバーをコンポーネントに散らさない
- 問題は動的生成（単純問題のみ固定データ）。答えは必ずロジックで計算し、問題データに答えをハードコードしない
- UI・問題文は日本語のみ

## ドメイン知識（コンボ計算の正解基準）

クイズの正解計算はこの定義に従う。**ここを間違えるとアプリの存在価値がなくなる**ので、変更時は必ず検算・テストすること。

- 総コンボ数: C(52,2) = **1326**
- ポケットペア: 1ランクにつき C(4,2) = **6**、13ランクで **78**
- スーテッド（非ペア）: 1ハンドタイプにつき **4**、78タイプで **312**
- オフスート（非ペア）: 1ハンドタイプにつき **12**、78タイプで **936**
- 非ペア2ランク合計（例: AK）: 4 + 12 = **16**
- デッドカード考慮: デッキからデッドカード（ボード等）を除いた残りカードで作れる該当ハンドの組み合わせ数を数える
  - 導出は **AK − AKs = AKo** の順（全体 → スーテッド → 引き算）で考える。例: ボード A♠ K♥ 2♦ のとき
    - AA: 残る A は3枚 → C(3,2) = **3**
    - AK 全体: 残る A 3枚 × 残る K 3枚 = **9**
    - AKs: A♠K♠ と A♥K♥ が死んでいる → 残るのは A♦K♦, A♣K♣ の **2**
    - AKo: 9 − 2 = **7**

表記規約: `s` = suited、`o` = offsuit、無印（例: AK）= 両方の合計。
