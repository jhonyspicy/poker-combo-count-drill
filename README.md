# poker-combo-count-drill

ポーカー（テキサスホールデム）の「コンボ数を数える力」を鍛えるクイズアプリです。
アプリの表示名は「**コンボカウントドリル**」。

公開URL: https://jhonyspicy.github.io/poker-combo-count-drill/

## コンセプト

- スマホ向けのWebアプリ。ログイン不要で誰でもすぐ遊べる
- クイズ形式でコンボ数のカウントを反復練習する
- 制限時間内に4択から正解を選ぶと次の問題へ進む
- 進むほど問題が難しくなり、制限時間が短くなる
- **連続で何問正解できたか**を競う

## ゲームルール

1. スタートすると1問目が表示される
2. 各問題には制限時間があり、4つの選択肢から回答する
3. 正解すると即座に次の問題へ（スコア +1）
4. **不正解、または時間切れでゲームオーバー**（ライフなし・一発アウト）
5. リザルト画面で今回のスコア・正解の解説・自己ベストを表示し、リトライできる

自己ベストは端末内（localStorage）にのみ保存する。サーバーやランキングは持たない。
保存する内容は **最高連続正解数・そのときの到達レベル・達成日時** の3つ。

## 画面構成

HashRouter による4画面構成。

| 画面 | 役割 |
|---|---|
| トップ | タイトル、自己ベスト表示、スタートボタン、遊び方ページへのリンク |
| プレイ | 問題・4択・残り時間の表示。ゲームの本体 |
| リザルト | 今回のスコア、ゲームオーバーになった問題の正解と解説、自己ベスト（更新演出あり）、リトライ・トップへのボタン |
| 遊び方 | ルール説明と、基礎知識のコンボ表（1326 / 78 / 312 / 936 など）を載せたチートシート |

### トップページの仕様

縦一列のシンプルな構成。上から順に：

1. **タイトルブロック** — スート記号（♠♥♦♣）＋「コンボカウントドリル」＋一行のキャッチコピー
2. **自己ベストカード** — 連続正解数を主役に、到達レベルと達成日を添える。記録がない初回訪問時はカードの代わりに「まずは1問目に挑戦！」のような案内文を表示する
3. **スタートボタン** — 画面下寄り（親指が届く位置）に最も大きく配置
4. **遊び方・コンボ表へのリンク** — スタートボタンの下に控えめに配置。最下部に「不正解・時間切れでゲームオーバー」の一行注記

前回のスコアはトップには表示しない（リザルト画面で見せる）。

## デザイン

- **ダーク基調で固定**（ライトモードなし）。ポーカールームらしい暗めの背景に、カードの白とスートの赤・黒が映える配色
- モバイルの縦画面・タップ操作を基準にレイアウトする（モバイルファースト）

## 問題タイプ

### v1 で実装するもの

#### 1. 単純問題（暗記系・固定データ）

答えが固定の基礎知識を問う。

| 問題 | 答え |
|---|---|
| 総コンボ数はいくつ？ | 1326 |
| ポケットペアは何コンボ？ | 78 |
| スーテッドは何コンボ？ | 312 |
| オフスートは何コンボ？ | 936 |
| AA（任意のペア）は何コンボ？ | 6 |
| AKs（任意のスーテッド）は何コンボ？ | 4 |
| AKo（任意のオフスート）は何コンボ？ | 12 |
| AK（任意の非ペア2ランク）は何コンボ？ | 16 |

#### 2. デッドカード問題（動的生成）

ボードをランダム生成し、デッドカードを考慮したコンボ数を問う。答えはロジックで計算する。

> 例: ボードに A♠ K♥ 2♦ が開かれたとき、相手は AKo を何コンボ持っていますか？（答え: 7）

- ボードはフロップ（3枚）／ターン（4枚）／リバー（5枚）のいずれか
- 高レベルほど枚数の多いボード（＝デッドカードが多く数えにくい問題）が出る
- 解説は **AK − AKs = AKo** の導出順で示す（まず全体 = 残りA × 残りK の 9、次に見えているスートから AKs = 2、引き算で AKo = 7）。実戦での数え方と同じ思考順序を身につけてもらうため

#### 3. レンジ表問題（動的生成）

13×13のレンジ表を表示し、ハイライトされたエリアの合計コンボ数を問う。
エリアはパターン（スーテッド全体・オフスート全体・ポケットペア・ブロードウェイペア・隣接スーテッド・Aを含むハンド・スーテッドコネクターなど）から選ばれ、答えはエリア内のタイプ数 ×（ペア6 / スーテッド4 / オフスート12）で計算する。

> 例: オフスートのエリアをハイライト → 78タイプ × 12 = 936 コンボ

### 将来の拡張候補（v1 には含めない）

- **順位当て問題**: 自分のハンドが同一ボード上で成立しうるハンドの中で何番目に強いかを問う（ハンド評価ロジックと「成立しうるハンド」の定義の詰めが必要）
- **レンジ vs コンボ問題**: 相手のレンジ表を表示し、自分に勝っているコンボ数を問う（レンジ表UIとプリセットレンジが必要）
- スコアのSNS共有（X など）
- 英語UI対応（i18n）

## レベルと難易度進行

習熟型のレベル進行。Lv1 は単純問題を全問正解でクリア、Lv2 以降は同レベルの問題を規定数（初期値20問）連続正解すると次のレベルへ進む。レベルに応じて (a) 出題タイプ、(b) 制限時間が変化する。

### 出題タイプの構成（1レベル＝1タイプ）

| レベル | 出題タイプ |
|---|---|
| Lv1 | 単純問題 |
| Lv2 | レンジ表問題 |
| Lv3 | デッドカード（フロップ） |
| Lv4 | デッドカード（ターン） |
| Lv5以降 | デッドカード（リバー） |

### 制限時間（初期案）

問題タイプごとの基準時間に、レベルに応じた短縮係数を掛ける。

- 基準時間: 単純 16秒 / レンジ表 24秒 / フロップ 30秒 / ターン 36秒 / リバー 40秒
- 短縮係数: レベルが1上がるごとに ×0.92（下限は基準時間の50%）

> ⚠️ 上記の数値（1レベルの問題数・基準時間・短縮係数）はすべて**プレイ感を見て調整する前提の初期値**。1か所の設定ファイルにまとめて定義し、簡単にチューニングできるようにする。

## 選択肢の生成

- 4択。正解1つ + 紛らわしい誤答3つを自動生成する
- 誤答は正解の近傍値や、ありがちな数え間違い（デッドカードの考慮漏れなど）から作る
