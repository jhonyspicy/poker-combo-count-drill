# poker-combo-count-drill

ポーカー（テキサスホールデム）の「コンボ数を数える力」を鍛えるクイズアプリです。
アプリの表示名は「**コンボカウントドリル**」。

公開URL: https://jhonyspicy.github.io/poker-combo-count-drill/

## コンセプト

- スマホ向けのWebアプリ。ログイン不要で誰でもすぐ遊べる
- クイズ形式でコンボ数のカウントを反復練習する
- 制限時間内に4択から正解を選ぶと次の問題へ進む
- 難易度（初級・中級・上級）をトップページで選び、同じ難易度の問題を無限に解き続ける
- 正解を重ねるほど制限時間が短くなっていく
- **連続で何問正解できたか**を難易度ごとに競う

## ゲームルール

1. トップページで難易度（初級編・中級編・上級編）を選ぶとスタートする
2. 各問題には制限時間があり、4つの選択肢から回答する
3. 正解すると即座に次の問題へ進む（連続正解 +1）。正解を重ねるほど制限時間が短くなる
4. **不正解、または時間切れでゲームオーバー**（ライフなし・一発アウト）
5. リザルト画面で今回のスコア・正解の解説・その難易度の自己ベストを表示し、同じ難易度でリトライできる

自己ベストは端末内（localStorage）にのみ保存する。サーバーやランキングは持たない。
保存する内容は **難易度ごとの最高連続正解数・達成日時**。

## 問題画面の3要素

問題画面は（回答の選択肢ボタンを除いて）次の3要素の出し分けで構成する。

| 要素 | 内容 |
|---|---|
| レンジ表 | 13×13 グリッド。出題エリアまたは相手のレンジをハイライトする |
| ボード | フェルト帯にボードのカードと自分のハンド（2枚）を表示する |
| 文章 | 問題文。キーワードを強調表示する |

## 難易度と問題タイプ

### 初級編 = レンジ表 + 文章

レンジ表の該当エリアをハイライトした上でコンボ数を問う基礎トレーニング。

- 単一ハンド: AKs / AKo / AA（ランクはランダム）は何コンボ？ → 4 / 12 / 6
- ハンドグループ: AK 合計(16)・Aを含むハンド(198)・スーテッド全体(312)・オフスート全体(936)・ポケットペア(78)・全ハンド(1326) など
- プリセットレンジ: 相手のレンジがレンジ表の通りのとき、相手が持ちうるコンボ数は？

答えは必ずエリア内のタイプ数 ×（ペア6 / スーテッド4 / オフスート12）で計算する。

### 中級編 = ボード + 文章

フロップ／ターン／リバー（ランダム）のボードと自分のハンドを見て、デッドカードを除いたコンボ数をカウントする実戦練習。

> 例: ボード A♠ K♥ 2♦・ハンド K♠ 9♥ のとき、相手が AKo を持っているコンボ数は？（答え: 4）

- 自分のハンドは**オープンレンジのハンドタイプから選出**し、スートをランダムに割り当てて具体化する
- 出題対象ハンド（相手のハンド）も**オープンレンジから選ぶ**。さらに、ボードまたは自分のハンドとランクが重なる（デッドカードが1枚以上絡む）ハンドに限定するため、答えは必ず素のコンボ数（ペア6 / スーテッド4 / オフスート12）より小さくなり、デッドカードを数えないと解けない
- 出題対象ハンドの残りコンボ数は必ず1以上（0コンボの問題は出さない）

解説は **AK − AKs = AKo** の導出順で示す（まず全体 = 残りA × 残りK、次にスーテッド、引き算でオフスート）。実戦での数え方と同じ思考順序を身につけてもらうため。

### 上級編 = レンジ表 + ボード + 文章

相手のプリセットレンジとボード・自分のハンドを見て、**何コンボに負けて（勝って）いるか**をカウントする総合演習。

> 例: 相手のレンジがレンジ表の通り、ボード A♠ K♥ 2♦・ハンド K♠ 9♥ のとき、あなたは何コンボに負けていますか？

- 答えはレンジ内の各ハンドをデッドカード除外で具体コンボに展開し、役評価ロジックで自分の役と比較して数える
- チョップ（引き分け）のコンボは勝ち・負けのどちらにも数えない
- 答えが 1〜60 に収まる問題だけを出題する（範囲外は再抽選）

### プリセットレンジ

相手のレンジは都度生成せず、あらかじめ定義したプリセットレンジからランダムに選ぶ。定義は `src/lib/presetRanges.ts`。

| レンジ名 | 内容 | 合計コンボ数 |
|---|---|---|
| オープンレンジ | ペア 77+ / A2s+ / K5s+ / Q9s+ / JTs / ATo+ / KJo+ | 216 |
| コールレンジ | ペア TT-44 / AQs-A7s・A5s-A4s / K9s+ / QTs+ / JTs / T9s / AJo | 118 |
| 3ベットレンジ | ペア JJ+ / AKs・A7s-A2s / K9s-K4s / Q9s+ / AJo+ / KJo+ / QJo | 160 |

## 画面構成

HashRouter による4画面構成。

| 画面 | 役割 |
|---|---|
| トップ | タイトル、難易度選択ボタン（3つ・各難易度の自己ベスト付き）、遊び方ページへのリンク |
| プレイ（`/play/:difficulty`） | 問題3要素・4択・残り時間の表示。ゲームの本体 |
| リザルト | 今回のスコアと難易度、ゲームオーバーになった問題の正解と解説、自己ベスト（更新演出あり）、同難易度リトライ・トップへのボタン |
| 遊び方 | ルール・難易度説明と、基礎知識のコンボ表（1326 / 78 / 312 / 936 など）を載せたチートシート |

### トップページの仕様

縦一列のシンプルな構成。上から順に：

1. **タイトルブロック** — スート記号（♠♥♦♣）＋「コンボカウントドリル」＋一行のキャッチコピー
2. **難易度選択ボタン** — 初級編・中級編・上級編の3ボタン。各ボタンに出題内容の一言説明と、その難易度の自己ベスト（記録がなければ「記録なし」）を添える
3. **遊び方・コンボ表へのリンク** — 控えめに配置。最下部に「不正解・時間切れでゲームオーバー」の一行注記

前回のスコアはトップには表示しない（リザルト画面で見せる）。

## デザイン

- **ダーク基調で固定**（ライトモードなし）。ポーカールームらしい暗めの背景に、カードの白とスートの4色が映える配色
- モバイルの縦画面・タップ操作を基準にレイアウトする（モバイルファースト）
- プレイ画面はヘッダー → タイマーバー → レンジ表パネル → フェルト帯（ボード｜あなた） → 問題文 → フィードバック → 2×2 選択肢の縦積み

## 制限時間

難易度別の基準時間に、連続正解数に応じた短縮係数を掛ける。

- 基準時間: 初級 12秒 / 中級 30秒 / 上級 45秒
- ボードあり問題はストリート別に加算（中級: ターン +6秒・リバー +10秒 / 上級: ターン +8秒・リバー +12秒）
- 短縮係数: 正解1回ごとに ×0.95（下限は基準時間の50%）

> ⚠️ 上記の数値はすべて**プレイ感を見て調整する前提の初期値**。`src/config/gameConfig.ts` にまとめて定義し、簡単にチューニングできるようにしている。

## 選択肢の生成

- 4択。正解1つ + 紛らわしい誤答3つを自動生成する
- 誤答は正解の近傍値や、ありがちな数え間違い（デッドカードの考慮漏れ、4/6/12/16 のハンドタイプ取り違えなど）から作る

## 将来の拡張候補

- **順位当て問題**: 自分のハンドが同一ボード上で成立しうるハンドの中で何番目に強いかを問う
- 文章のみの問題タイプ（データモデル上は3要素とも省略可能で拡張余地あり）
- スコアのSNS共有（X など）
- 英語UI対応（i18n）

## 技術スタック

- Vite + React 19 + TypeScript
